Задание №2 «Всегда в центре»

По ширине страницы, если блок имеет фиксированную ширину

Блок с фиксированной шириной расположен по центру

Плюсы:

  • кросс-браузерно;
  • просто для написания и понимания.

Минусы:

  • нужно следить за шириной внешнего блока, если он будет меньше, то левая часть блока скроется;
  • блок вырывается из потока;
  • отступ нужно считать вручную.

По ширине страницы, если блок имеет ширину, которая подстраивается под размер содержимого



Ширина блока подстраивается по контент

Плюсы:

  • кросс-браузерно.

Минусы:

  • блок старается сделаться шириной в половину ширины родителя;
  • блок вырывается из потока.

По ширине страницы, если блок имеет ширину, которая подстраивается под размер содержимого (вариант без абсолютного позиционирования)

Ширина блока подстраивается по контент

Плюсы:

  • кросс-браузерно;
  • ширина подстраивается дополнительно и под родителя;
  • не вырывается из потока.

Минусы:

  • для 6-7 ишаков нужно эмулировать инлайн-блок.

По ширине страницы, если блок имеет фиксированную ширину (вариант без абсолютного позиционирования)

Ширина блока известна

Плюсы:

  • кросс-браузерно;
  • блок не вылетает в не-скролящиеся области;
  • не вырывается из потока.

Минусы:

  • ИМХО, нет =).

По высоте страницы, если блок имеет фиксированную высоту

Размеры блока известны

Плюсы:

  • работает, и это не может не радовать;
  • кросс-браузерно, насколько мне известно;
  • отлично подходит для попапов и модалок.

Минусы:

  • вырывается из потока, хотя минус-ли?

По высоте страницы, если высота зависит от контента

Высота блока считается исходя из контента

Плюсы:

  • появляется скролл;
  • можно зафигачить и по горизонтальному центру;
  • подходит для модалок с хренпоймикаким контентом.

Минусы:

  • нужен отдельный врапер;
  • нужен хелпер;
  • не работает в ИЕ7.
